## 1. MAPA DE INTERFACES VITALES (SANEADO)

El sistema **PTC Virtual** unifica sus comunicaciones bajo el protocolo **Vital IP**, eliminando la complejidad de gateways de protocolo propietarios.

### 1.1 Matriz de Interfaces:

| ID | Origen | Destino | Protocolo | Tipo |
|:---|:---|:---|:---|:---|
| **IF-V-01** | OBC (Locomotora) | Servidor PTC | **Vital IP / TETRA** | Vital |
| **IF-V-02** | EOT (Cola) | OBC | **Vital IP / Radio** | Vital |
| **IF-V-03** | CTC (CCO) | Servidor PTC | **TCP/IP Seguro** | Operativa |
| **IF-V-04** | Servidor PTC | ENCE (5 Est) | **Vital IP / Fibra** | Vital |
| **IF-V-05** | CCO La Dorada | CCO FENOCO | **Stop & Switch** | Procedimental |

---

## 2. 🔍 AUDITORÍA DE PURGA (RECHAZO DE GATEWAYS)

Se ha realizado una limpieza extrema de la v5.0 (ICD):
- ✅ **ELIMINADO:** Gateways de conversión para PTC/PTC VIRTUAL. El sistema opera nativamente en **PTC Virtual**.
- ✅ **ELIMINADO:** Interfaz Red Vital IP / TETRA. El canal de datos se unifica en **TETRA**.
- ✅ **ELIMINADO:** Protocolos SCI-CC-A (ADIF) no aplicables.
- ✅ **ELIMINADO:** Referencia a "Bloque Integración Multi-sistema" de alto costo.

---

## 🏗️ ESPECIFICACIÓN TÉCNICA VITAL IP
- **Capa de Transporte:** TCP/UDP sobre red MPLS de Fibra Óptica.
- **Seguridad Vital:** Implementación de capas de chequeo CRC32, Timestamps de 64 bits y números de secuencia para evitar ataques de repetición o pérdida de integridad.
- **Latencia:** Máximo 150ms punta a punta (OBC ↔ Servidor).

---

## ✅ CONCLUSIONES:

Este ICD define un entorno de integración simplificado y soberano. Al eliminar los gateways propietarios, se reduce el riesgo de integración y se optimiza el costo de licencias de terceros.
